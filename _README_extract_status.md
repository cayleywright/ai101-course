## Heads up — paste fix

The chat client auto-converted some JavaScript dot notation into Markdown
hyperlinks during paste (e.g. `el.style.animation` came through as
`[el.style](http://el.style).animation`). I restored these to plain JS
when saving so the scripts will actually run. Specific patterns fixed:

- `[el.style](...).animation` → `el.style.animation`
- `[label.style](...).color` → `label.style.color`
- `[card.style](...).borderColor` → `card.style.borderColor`
- `[chip.style](...).cssText` → `chip.style.cssText`
- `[circle.style](...).cursor` → `circle.style.cursor`
- `[t.cx](...)`, `[t.cy](...)`, `[t.name](...)`, `[l.name](...)` etc. — fixed to plain dot notation
- `[node.direct](...)` → `node.direct`
- `[s.feedback](...).right`/`.wrong[idx]` → `s.feedback.right`/`.wrong[idx]`
- `[rect.top](...)` → `rect.top`
- CSS selectors `.[ts-quiz-next.show](...)`, `.[ts-arch-subrow-wrap.open](...)`,
  `.[ts-arch-card-wrap.open](...)` — restored to proper compound selectors
